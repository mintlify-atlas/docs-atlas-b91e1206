This guide takes you from zero to a running Pyxel game. You'll learn the core game loop pattern, the recommended class-based structure, and how to run your scripts.

## Minimal game

Every Pyxel application follows the same structure: initialize the window, define `update` and `draw` functions, then call `run`.

```python hello.py
import pyxel

pyxel.init(160, 120)

def update():
    if pyxel.btnp(pyxel.KEY_Q):
        pyxel.quit()

def draw():
    pyxel.cls(0)
    pyxel.rect(10, 10, 20, 20, 11)

pyxel.run(update, draw)
```

- `pyxel.init(width, height)` — sets the window size in pixels.
- `update()` — called once per frame to handle input and game logic.
- `draw()` — called once per frame to render the screen.
- `pyxel.run(update, draw)` — starts the game loop. This call blocks until the application exits.

## The update/draw loop

Pyxel runs your game at a fixed frame rate (default 30 FPS). Each frame:

1. `update()` runs first — process input, move objects, update state.
2. `draw()` runs second — clear the screen with `cls()`, then draw everything.

<Note>
  Always call `pyxel.cls()` at the start of `draw()`. Without it, each frame is drawn on top of the previous one.
</Note>

## Class-based pattern (recommended)

For real applications, wrapping your code in a class keeps state organized and avoids global variables. This is the pattern used in all official Pyxel examples.

The following is the actual source of `01_hello_pyxel.py`:

```python 01_hello_pyxel.py
import pyxel


class App:
    def __init__(self):
        pyxel.init(160, 120, title="Hello Pyxel")
        pyxel.images[0].load(0, 0, "assets/pyxel_logo_38x16.png")
        pyxel.run(self.update, self.draw)

    def update(self):
        if pyxel.btnp(pyxel.KEY_Q):
            pyxel.quit()

    def draw(self):
        pyxel.cls(0)
        pyxel.text(55, 41, "Hello, Pyxel!", pyxel.frame_count % 16)
        pyxel.blt(61, 66, 0, 0, 0, 38, 16)


App()
```

Key points:

- `pyxel.init(..., title="Hello Pyxel")` — the optional `title` parameter sets the window title.
- `pyxel.images[0].load(0, 0, "path.png")` — loads an image into image bank 0 at position (0, 0).
- `pyxel.frame_count % 16` — cycles through all 16 palette colors, creating an animated text color.
- `pyxel.blt(x, y, img, u, v, w, h)` — draws a region from an image bank to the screen.
- `App()` at the bottom instantiates the class, which triggers `__init__` and starts the loop via `run`.

## Static graphics with `show()`

If you only need to display something without animation, use `show()` instead of `run()`. This draws once and keeps the window open until the user closes it.

```python static.py
import pyxel

pyxel.init(120, 120)
pyxel.cls(1)
pyxel.circb(60, 60, 40, 7)
pyxel.show()
```

`show()` is useful for quick visual tests and static artwork — no update loop needed.

## Running your script

<Steps>
  <Step title="Write your script">
    Create a Python file (e.g., `game.py`) with your Pyxel code.
  </Step>
  <Step title="Run it">
    You can run Pyxel scripts two ways:

    <CodeGroup>

    ```bash pyxel run
    pyxel run game.py
    ```

    ```bash python
    python game.py
    ```

    </CodeGroup>

    Both work. `pyxel run` is the recommended approach — it handles environment setup automatically.
  </Step>
  <Step title="Auto-reload during development">
    Use `pyxel watch` to automatically restart your script whenever you save changes:

    ```bash
    pyxel watch . game.py
    ```

    Stop watching with `Ctrl+C` (or `Cmd+C` on Mac).
  </Step>
</Steps>

## Special key shortcuts

These shortcuts are available in any running Pyxel application:

| Shortcut | Action |
| --- | --- |
| `Esc` | Quit the application |
| `Alt+R` (or `A+B+X+Y+BACK` on gamepad) | Reset the application |
| `Alt+1` | Save a screenshot to the desktop |
| `Alt+2` | Reset the screen capture recording start time |
| `Alt+3` | Save a screen capture video to the desktop (up to 10 seconds) |
| `Alt+8` (or `A+B+X+Y+DL` on gamepad) | Toggle screen scaling between maximum and integer |
| `Alt+9` (or `A+B+X+Y+DR` on gamepad) | Switch screen mode (Crisp / Smooth / Retro) |
| `Alt+0` (or `A+B+X+Y+DU` on gamepad) | Toggle performance monitor (FPS / update time / draw time) |
| `Alt+Enter` (or `A+B+X+Y+DD` on gamepad) | Toggle fullscreen |
| `Shift+Alt+1/2/3` | Save image bank 0, 1, or 2 to the desktop |
| `Shift+Alt+0` | Save the current color palette to the desktop |

<Tip>
  On Mac, use `Option` in place of `Alt` for all shortcuts above.
</Tip>

## Explore examples

Copy the built-in examples to your current directory to see more complete games:

```bash
pyxel copy_examples
cd pyxel_examples
pyxel run 01_hello_pyxel.py
pyxel run 02_jump_game.py
```

You can also browse and run examples in the browser at [Pyxel Showcase](https://kitao.github.io/pyxel/web/showcase/).
