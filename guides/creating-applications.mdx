## The init/run/update/draw pattern

Every Pyxel application follows the same lifecycle:

1. Call `pyxel.init()` to configure the window.
2. Define `update()` and `draw()` functions.
3. Call `pyxel.run(update, draw)` to start the loop.

```python
import pyxel

pyxel.init(160, 120)

def update():
    if pyxel.btnp(pyxel.KEY_Q):
        pyxel.quit()

def draw():
    pyxel.cls(0)
    pyxel.rect(10, 10, 20, 20, 11)

pyxel.run(update, draw)
```

`update` is called once per frame to process game logic. `draw` handles rendering and may be skipped if the engine cannot keep up with the target frame rate.

## pyxel.init() parameters

`pyxel.init(width, height, ...)` initializes the window and must be called before any other Pyxel API.

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `width` | int | — | Screen width in pixels (required) |
| `height` | int | — | Screen height in pixels (required) |
| `title` | str | `"Pyxel"` | Window title bar text |
| `fps` | int | `30` | Target frames per second |
| `quit_key` | int | `KEY_Q` | Key that quits the app (`KEY_NONE` to disable) |
| `display_scale` | int | auto | Integer scale factor for the window |
| `capture_scale` | int | auto | Scale factor used when saving screenshots |
| `capture_sec` | int | `10` | Max duration (seconds) for screen capture video |
| `headless` | bool | `False` | Run without creating a window (for testing) |

```python
pyxel.init(
    256, 192,
    title="My Game",
    fps=60,
    quit_key=pyxel.KEY_ESCAPE,
    display_scale=3,
    capture_scale=2,
    capture_sec=15,
)
```

## Class-based approach

For anything beyond a small script, wrapping your app in a class keeps state organized and avoids global variables.

```python
import pyxel

class App:
    def __init__(self):
        pyxel.init(160, 120, title="Pyxel Jump")
        pyxel.load("assets/jump_game.pyxres")

        self.score = 0
        self.player_x = 72
        self.player_y = -16
        self.player_dy = 0
        self.is_alive = True

        pyxel.playm(0, loop=True)
        pyxel.run(self.update, self.draw)

    def update(self):
        if pyxel.btnp(pyxel.KEY_Q):
            pyxel.quit()
        self.player_y += self.player_dy
        self.player_dy = min(self.player_dy + 1, 8)

    def draw(self):
        pyxel.cls(12)
        pyxel.text(5, 4, f"SCORE {self.score:>4}", 7)

App()
```

<Tip>
`pyxel.run()` blocks until the app exits, so call it at the end of `__init__`. The `App()` call at the bottom of the file starts the application.
</Tip>

Here is a more complex class-based example from the Pyxel platformer, showing how entities can have their own `update` and `draw` methods that the main `App` calls:

```python
import pyxel

TRANSPARENT_COLOR = 2

class Player:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        self.dx = 0
        self.dy = 0
        self.direction = 1
        self.is_falling = False

    def update(self):
        if pyxel.btn(pyxel.KEY_LEFT):
            self.dx = -2
            self.direction = -1
        if pyxel.btn(pyxel.KEY_RIGHT):
            self.dx = 2
            self.direction = 1

        self.dy = min(self.dy + 1, 3)

        if pyxel.btnp(pyxel.KEY_SPACE):
            self.dy = -6
            pyxel.play(3, 8)

        self.dx = int(self.dx * 0.8)

    def draw(self):
        u = (2 if self.is_falling else pyxel.frame_count // 3 % 2) * 8
        w = 8 if self.direction > 0 else -8
        pyxel.blt(self.x, self.y, 0, u, 16, w, 8, TRANSPARENT_COLOR)


class App:
    def __init__(self):
        pyxel.init(128, 128, title="Pyxel Platformer")
        pyxel.load("assets/platformer.pyxres")
        self.player = Player(0, 0)
        pyxel.playm(0, loop=True)
        pyxel.run(self.update, self.draw)

    def update(self):
        if pyxel.btn(pyxel.KEY_Q):
            pyxel.quit()
        self.player.update()

    def draw(self):
        pyxel.cls(0)
        self.player.draw()

App()
```

## Static scenes with pyxel.show()

For graphics that do not require an animation loop, use `pyxel.show()` instead of `pyxel.run()`. It renders the current frame and waits for the window to be closed.

```python
import pyxel

pyxel.init(120, 120)
pyxel.cls(1)
pyxel.circb(60, 60, 40, 7)
pyxel.show()
```

<Note>
`pyxel.show()` is useful for one-shot visualizations, quick previews, and debugging. It does not call any update or draw callbacks.
</Note>

## Manual frame control with pyxel.flip()

`pyxel.flip()` advances one frame: it calls the OS event loop, draws the current state to the screen, and returns. Use it when you want to drive the frame loop yourself instead of delegating to `pyxel.run()`.

```python
import pyxel

pyxel.init(160, 120)

while True:
    pyxel.cls(0)
    pyxel.rect(pyxel.frame_count % 160, 50, 8, 8, 11)
    pyxel.flip()
```

<Warning>
`pyxel.flip()` is not available in the web version of Pyxel. If you need cross-platform compatibility, use `pyxel.run()` instead.
</Warning>

## pyxel.quit() and pyxel.reset()

- `pyxel.quit()` — exits the application immediately. Commonly called from `update()` in response to a key press.
- `pyxel.reset()` — restarts the application from the beginning, equivalent to pressing `Alt+R` at runtime.

```python
def update():
    if pyxel.btnp(pyxel.KEY_Q):
        pyxel.quit()

    if pyxel.btnp(pyxel.KEY_R):
        pyxel.reset()
```

## Runtime keyboard shortcuts

The following shortcuts are available while any Pyxel application is running:

| Shortcut | Action |
| --- | --- |
| `Esc` | Quit the application |
| `Alt+R` | Reset the application |
| `Alt+1` | Save a screenshot to the desktop |
| `Alt+2` | Reset the screen capture recording start time |
| `Alt+3` | Save a screen capture video to the desktop (up to the configured `capture_sec`) |
| `Alt+8` | Toggle screen scaling between maximum and integer |
| `Alt+9` | Switch screen mode (Crisp / Smooth / Retro) |
| `Alt+0` | Toggle performance monitor (FPS, `update` time, `draw` time) |
| `Alt+Enter` | Toggle fullscreen |
| `Shift+Alt+0` | Save the current color palette to the desktop |
| `Shift+Alt+1` | Save image bank 0 to the desktop |
| `Shift+Alt+2` | Save image bank 1 to the desktop |
| `Shift+Alt+3` | Save image bank 2 to the desktop |

On macOS, use `Option` in place of `Alt`. Gamepad equivalents for several shortcuts are also available (e.g., `A+B+X+Y+BACK` to reset).

## Headless mode for testing

Pass `headless=True` to `pyxel.init()` to run without opening a window. This is useful for automated tests, CI pipelines, and scripts that only need the Pyxel API without a display.

```python
import pyxel

pyxel.init(160, 120, headless=True)

# Run the game loop for a fixed number of frames
for _ in range(60):
    pyxel.flip()

# Verify game state without rendering
assert pyxel.frame_count == 60
```

<Note>
In headless mode, `draw()` is still called but nothing is displayed. All input functions return their default values (no keys pressed, no mouse movement).
</Note>

## Running your script

<Steps>
  <Step title="Run directly with Python">
    ```bash
    python my_game.py
    ```
  </Step>
  <Step title="Run with the pyxel command">
    ```bash
    pyxel run my_game.py
    ```
  </Step>
  <Step title="Watch for changes and auto-reload">
    ```bash
    pyxel watch . my_game.py
    ```
    Press `Ctrl+C` to stop watching.
